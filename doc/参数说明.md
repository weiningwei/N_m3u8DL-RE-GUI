# N_m3u8DL-RE 命令行参数说明（GUI 开发参考）

> 整理自上游仓库 [nilaoda/N_m3u8DL-RE](https://github.com/nilaoda/N_m3u8DL-RE) 的 `README.md`「命令行参数」章节，
> 供本 GUI（N_m3u8DL-RE-GUI，Rust + iced）后续开发对照使用。
>
> **权威映射源**：参数 → GUI 控件的唯一实现位置是 [`src/args.rs`](../src/args.rs) 的 `build_args()`；
> 参数预览的拼装逻辑在 [`src/app.rs`](../src/app.rs) 的 `command_preview()`。本文档是其结构化整理版，
> 若与代码不一致，以 `build_args()` 为准。

---

## 约定

- **GUI 状态**：`已实现` = 已在 `build_args()` 中拼装；`未实现` = 上游支持但 GUI 暂未提供控件。
- **位置参数 `<input>`**：下载地址或本地文件路径，对应「基本」页的“下载地址/文件”输入框。
- **布尔开关输出策略**：`src/args.rs` 的 `bool_opt()` 仅在“值与 RE 默认值不同”时才输出该参数
  （例如 `--del-after-done` 默认 `true`，用户取消勾选时才输出 `--del-after-done false`）。
- 所有参数均在「与默认值不同 / 文本框非空」时才出现在预览命令行中。

## 附录 A：复合参数（sub-option）语法

### `-M, --mux-after-done`
以 `:` 分隔，支持：`format=FORMAT`(mkv/mp4)、`muxer=MUXER`(ffmpeg/mkvmerge，默认 ffmpeg)、
`bin_path=PATH`、`skip_sub=BOOL`、`keep=BOOL`(保留源文件)。
示例：`-M format=mp4` / `-M format=mkv:muxer=mkvmerge`。

### `--mux-import`
以 `:` 分隔，支持：`path=PATH`(必填)、`lang=CODE`、`name=NAME`。
示例：`--mux-import path=zh-Hans.srt:lang=chi:name="中文 (简体)"`。

### `-sv / -sa / -ss`（select-video / select-audio / select-subtitle）
以 `:` 分隔，支持：`id=REGEX` `lang=REGEX` `name=REGEX` `codecs=REGEX` `res=REGEX`
`frame=REGEX` `segsMin=number` `segsMax=number` `ch=REGEX` `range=REGEX` `url=REGEX`
`plistDurMin=hms` `plistDurMax=hms` `for=FOR`(best[number]/worst[number]/all，默认 best)。
示例：`-sv res="3840*":codecs=hvc1:for=best` / `-sa lang="ja|en":for=best2`。

### `--custom-range`
示例：`0-10`(第 0~10 共 11 个) / `10-`(从 10 开始) / `-99`(前 100 个) / `05:00-20:00`(第 5~20 分钟)。

### `--save-pattern` 支持的变量
`<SaveName>` `<Id>` `<Codecs>` `<Language>` `<Resolution>` `<Bandwidth>` `<MediaType>`
`<Channels>` `<FrameRate>` `<VideoRange>` `<GroupId>` `<Ext>`。
用于多路同类型流避免文件名冲突；不使用时 RE 会用流元数据自动去重。

## 附录 B：GUI 开发注意事项

- **参数拼装唯一源**：任何“参数 ↔ 控件”的对应关系以 [`src/args.rs`](../src/args.rs) 的 `build_args()` 为准；
  新增/修改参数务必同步更新该函数与对应 `App` 字段、`Message`、`ui.rs` 控件。
- **命令行预览**：[`src/app.rs`](../src/app.rs) 的 `command_preview()` 负责拼装
  `exe + 输入 + 参数` 的完整字符串，底部「命令预览」实时显示，「复制命令」按钮将其写入剪贴板。
- **代理“不使用”模式**：RE 没有显式禁用代理的命令行选项，GUI 选择“不使用代理”时当前不输出任何代理参数，
  依赖 RE 默认行为；若需显式效果，可能需要 `--no-proxy` 之类上游未提供的扩展。
- **布尔开关默认值**：`bool_opt()` 在与 RE 默认值相同时静默省略，可在「命令预览」中直接验证输出是否符合预期。
