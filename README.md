# N_m3u8DL-RE-GUI

> 让强大的 `N_m3u8DL-RE` 命令行下载器，也能「点点鼠标」就用起来。

[![Version](https://img.shields.io/badge/version-1.0.0-blue)](CHANGELOG.md)
[![Platform](https://img.shields.io/badge/platform-Windows%20%2F%20Cross--platform-lightgrey)](https://github.com/weiningwei/N_m3u8DL-RE-GUI)
[![Built with](https://img.shields.io/badge/built%20with-Rust%20%2B%20iced-9cf)](https://github.com/iced-rs/iced)
[![License](https://img.shields.io/badge/license-see%20LICENSE-green)](LICENSE)

`N_m3u8DL-RE-GUI` 是 [`N_m3u8DL-RE`](doc/README.md#命令行参数) 的**第三方图形界面**。
它把繁琐的命令行参数变成直观的标签页表单，自动拼好命令并启动下载器，
让你无需记忆参数、无需手敲命令，也能轻松下载、解密与合并 DASH / HLS / MSS 流媒体。

它**原生、轻量、跨平台**（基于 Rust + [iced](https://github.com/iced-rs/iced) 0.14），
本身只是一个「参数生成器 + 启动器」——真正的下载工作仍由上游 `N_m3u8DL-RE` 完成，
因此完整能力与上游保持一致。

## 功能特性

GUI 将 `N_m3u8DL-RE` 的主要命令行选项按分类整理到多个标签页，所见即所得：

| 标签页 | 主要内容 |
| --- | --- |
| **基本** | 可执行文件路径、下载地址/文件、保存目录/文件名/命名模板、临时目录、日志文件路径、BaseURL、线程数（默认 8）、重试次数（默认 10）、超时（默认 10s）、请求头、代理（系统/不使用/自定义）、限速、并发下载、界面语言、ffmpeg 路径、解密引擎与工具路径 |
| **流选择** | 自动选择最佳轨道、按表达式选择/去除视频/音频/字幕（`-sv`/`-sa`/`-ss`/`-dv`/`-da`/`-ds`）、仅选字幕、字幕格式（SRT/VTT） |
| **解密** | 解密密钥 `--key`、密钥文件 `--key-text-file`、自定义 HLS 加密方式、HLS 解密 KEY/IV、实时解密 MP4 分片 |
| **直播** | 点播方式录制、实时合并、保留分片、管道混流、VTT 字幕修正、录制时长限制、列表刷新间隔、首次获取分片数等 |
| **高级** | URL 参数附加、分片数量检测、输出 meta json、日志级别、urlprocessor 参数、分片范围、延时开始、广告关键字、混流参数 `-M`、外部媒体导入、二进制合并、各种开关 |
| **日志** | 实时运行日志、复制命令、打开输出文件夹、开始下载按钮 |

## 核心亮点

- **零命令记忆**：所有参数可视化，底部「命令预览」实时生成最终命令行，可一键复制手动执行。
- **实时日志不卡顿**：捕获模式下日志随进程流式刷新，中文正常显示（自动适配 UTF-8 / GBK），
  不再出现乱码或「刷不出来」。
- **两种运行模式**：默认捕获日志到内置面板；也可勾选「外部控制台窗口」，像旧版 SimpleG 那样弹出独立控制台运行。
- **开箱即用**：首次启动即带有合理的默认值（线程数 8、重试 10、超时 10s），放好 `N_m3u8DL-RE` 即可直接下载。
- **智能探测**：未手动指定时，自动在「用户路径 → 同目录 → 系统 PATH」中查找 `N_m3u8DL-RE` 与 `ffmpeg`。
- **偏好记忆**：记住可执行文件、保存目录、代理、请求头、主题、外部控制台等设置。
- **深色模式**：跟随 Windows 系统主题自动切换浅色 / 深色。
- **键盘流**：ESC 退出、日志区 Ctrl+C 复制、Tab / Shift+Tab 切换焦点、回车即开始下载。
- **中文友好**：界面使用微软雅黑，捕获日志正确解码中文，告别乱码。

## 环境要求

1. **Rust 工具链**：edition 2024，建议使用 **Rust 1.85 或更高版本**（通过
   [rustup](https://rustup.rs/) 安装）。
2. **N_m3u8DL-RE 可执行文件**：本 GUI 本身不包含下载器，需自行准备官方构建的
   `N_m3u8DL-RE`（Windows 下为 `N_m3u8DL-RE.exe`）。
   - 可放在与 GUI 可执行文件相同的目录，或加入系统 `PATH`，GUI 会自动探测；
   - 也可在「基本」页手动指定其完整路径。

## 构建

```bash
# 调试构建（开发用）
cargo build

# 发布构建（推荐，体积小、运行快）
cargo build --release
```

构建产物：

- 调试：`target/debug/n_m3u8dl-re-gui.exe`
- 发布：`target/release/n_m3u8dl-re-gui.exe`

Windows 下首次构建会编译 `iced` / `wgpu` 等依赖，耗时较长，请耐心等待。

---

## 运行

```bash
# 直接运行（调试构建）
cargo run

# 或运行已构建的可执行文件
./target/release/n_m3u8dl-re-gui.exe
```

首次使用时，请在「基本」页填写 `N_m3u8DL-RE.exe` 的路径与下载地址，然后点击
「开始下载」。运行过程中可切换到「日志」页查看输出。

## 使用说明

1. **指定下载器**：在「基本」页设置 `N_m3u8DL-RE.exe` 路径（或在 GUI 同目录 / PATH 中放置它）。
2. **填写下载地址**：支持在线 URL，也支持本地 `.m3u8` / `.mpd` 文件路径。
3. **按需配置选项**：在对应标签页勾选/填写需要的参数；未填写的可选参数不会出现在命令行中。
   其中线程数（默认 8）、重试次数（默认 10）、HTTP 超时（默认 10s）已预填并会始终传入。
4. **命令预览**：底部文本框始终显示最终命令行，确认无误后可点「复制命令」手动执行。
5. **开始下载**：点击「开始下载」；GUI 会在内部捕获日志并展示在「日志」页（详见[配置存储](#配置存储)）。
   若勾选了「使用外部控制台窗口」，则会弹出独立控制台运行（不捕获日志）。

> 各选项对应的 `N_m3u8DL-RE` 命令行参数与含义，见仓库内文档：
> - [`doc/参数说明.md`](doc/参数说明.md) —— 按 GUI 标签页整理的参数对照（含实现状态，供开发参考）
> - [`doc/README.md`](doc/README.md#命令行参数) —— `N_m3u8DL-RE` 上游官方参数文档
>
> 本 GUI 本质上就是这些命令行参数的可视化生成器与启动器。

## 配置存储

GUI 会记住少量用户偏好（可执行文件路径、保存目录、代理地址、请求头、外部控制台开关、主题、ffmpeg 路径），
保存为 JSON 配置文件，位于**可执行文件所在目录**下的 `config/config.json`
（即与 GUI 的 `.exe` 同级的 `config` 子目录中）：

- Windows：`N_m3u8DL-RE-GUI.exe` 所在目录 `\config\config.json`
- Linux / macOS：可执行文件所在目录 `/config/config.json`

下载日志文件默认写入**可执行文件所在目录**下的 `log/n_m3u8dl-re-gui.log`
（可在「基本」页的“日志文件路径”中修改）。

删除配置（及日志）文件即恢复默认设置。

## 项目结构

```
src/
├── main.rs      # 程序入口，iced 应用装配与字体加载
├── app.rs       # 应用状态 App、消息枚举 Message、参数拼装辅助
├── args.rs      # 将 App 状态转换为 N_m3u8DL-RE 命令行参数
├── config.rs    # 配置的加载与持久化（JSON）
├── detect.rs    # 定位 N_m3u8DL-RE 可执行文件
├── runner.rs    # 启动子进程、捕获输出、日志订阅
└── ui.rs        # 各标签页的界面布局
```
